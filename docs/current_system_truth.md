# Current System Truth

Discovery Intelligence today is a session-aware molecular decision-support workbench. It lets users upload data, run prioritization workflows, review candidates, reopen sessions, compare runs, and carry forward workspace feedback. It is a real product surface with operational scaffolding, not just a modeling script.

What it is good at today is workflow continuity, reviewability, session persistence, decision traceability, and increasingly honest session rendering. The strongest current scientific truth is that it can produce recommendation lists from chemistry-based representations with explicit session context, target-aware caution signals, and clearer regression-vs-classification semantics. It now also creates first-class proposed claims, claim-linked proposed experiment requests, human-entered observed experiment-result records, bounded belief-update records, and a target-scoped BeliefState summary that tracks the current support picture without implying final truth or automatic learning. That BeliefState layer now distinguishes mostly proposed vs mostly accepted support more explicitly, carries a cautious read-across alignment summary for the current session, depends more directly on explicit human acceptance or rejection of belief updates, treats superseded support changes as visible historical records rather than current active support, surfaces compact current-vs-historical chronology summaries across Discovery, Dashboard, Sessions, and session comparison, groups support chronology more clearly at the individual claim level, and now adds a lightweight cross-session claim read-across layer so a claim can be surfaced as continuity-aligned, new-context, weakly related, or lacking prior claim context without implying exact claim identity. That claim read-across layer now also weights prior continuity by whether it is backed by posture-governing accepted support, only tentative active support, only historical support, sparse unsupported prior context, no useful prior governed context, or contested/degraded prior support that should only be reused cautiously. The scientific-core path is now also better at separating support existence from support usefulness: observed-result summaries can distinguish stronger bounded interpretation, numeric-rule-based but still limited interpretation, screening/provisional caution, context-limited interpretation, and unresolved interpretation; BeliefUpdate records now carry an explicit support-quality layer saying whether active support is decision-useful, still limited, context-limited, or weak/unresolved; and BeliefState plus session-level decision summaries now roll those support-quality distinctions up into a clearer “current support quality” picture. On top of that, the system now carries a separate governed-support-posture layer that says whether support is accepted and posture-governing, accepted but still limited-weight, active but still tentative, or historical-only after supersession. It now also carries bounded contradiction/degradation semantics so mixed accepted updates can weaken present posture without being framed as final disproof: current support can now be surfaced as coherent, contested, degraded, contested-and-degraded, historically stronger than the present, or lacking coherent current support at all. Those semantics also drive a lightweight governed-reuse layer saying whether support is strongly reusable, selectively reusable, weakly reusable, contradiction-limited, historical-only for reuse, or not yet suitable for strong governed reuse. The newer broader governed-reuse slices now keep local support separate from broader governed reuse: claim summaries, target-scoped belief-state summaries, session-level decision summaries, and session comparison can now say whether broader reuse is strong, selective, contradiction-limited, historical-only, or still only locally meaningful; whether broader continuity is coherent, selective, contested, historical-heavy, or absent; whether continuity is local-only, context-only, selective, contradiction-limited, historical-heavy, or coherent enough to count as a later broader governed promotion candidate; and whether that candidate is actually promotable, selectively promotable, blocked, downgraded, quarantined, or not yet a governed promotion candidate at all. The system can now also surface bounded promotion stability and compact promotion block reasons, such as contradiction-heavy history, degraded present posture, historical-heavy continuity, or context-only meaning. The trust and governance substrate now extends beyond claims: evidence activation rolls up explicit source-class and provenance-confidence semantics; upload-derived evidence remains local-only by default; repeated rows or repeated uploads no longer increase evidence support strength by count; claims persist lightweight governed review records; and that persisted review ledger now also captures target-scoped belief-state posture, continuity-cluster posture, and session-family carryover posture with approval, block, defer, downgrade, quarantine, reversal history, explicit reviewer-attributed manual actions, concise reviewer notes, reopen/revise transitions, carryover-effect summaries, and now explicit reviewer-attributed manual actions alongside derived snapshots. Claims, target-scoped belief-state summaries, session-level decision summaries, and comparison rows now inherit support-basis composition, support-quality interpretation, governed posture, coherence, reuse, broader-reuse, continuity-cluster posture, promotion-candidate posture, promotion stability, promotion-gate status, promotion block-reason signals, compact source-class framing, persisted governed-review history, promotion-audit summaries, explicit anti-poisoning guardrails, and a compact distinction between derived posture, manual reviewed posture, effective current posture, and reviewer rationale context. A specific claim, belief-state picture, continuity cluster, or session-family carryover picture can now be surfaced as locally meaningful but not broadly governing, selectively reusable across related claims, contradiction-limited for broader reuse, mainly historical context for later reuse thinking, coherent enough to be a bounded promotion candidate later, blocked from promotion under current bounded rules, or still useful locally while broader influence remains deferred because provenance, source class, continuity, or higher-layer review posture is too weak. It can also now be surfaced as manually approved, blocked, deferred, downgraded, quarantined, reopened, or revised without erasing the underlying derived posture, and prior stronger reviewed posture can remain visible as superseded history rather than being silently overwritten. The product now also has a bounded reviewer workflow shell on top of that ledger: a governance inbox, per-item inspection context, reviewer-attribution summaries, cross-session carryover attention grouping, concise reviewer note context, reopen/revise visibility, and a dedicated reviewer surface make manual governance more operational without changing the product into an enterprise workflow suite. Claim usefulness and session posture now also depend more explicitly on whether current support is actually posture-governing versus merely present on paper, whether contradiction pressure materially weakens the present posture, whether broader continuity remains strong enough to matter now, and whether that continuity should remain context-only, blocked, downgraded, quarantined, or selectively promotable rather than silently reading as broadly promotable. Experiment requests now also inherit a compact bounded-intent layer so they can be surfaced as confirmatory, strengthening, clarifying, fresh-evidence, or exploratory follow-up depending on the linked claim picture rather than looking interchangeable. Discovery, Dashboard, Sessions, comparison, and the governance inbox now also carry a compact scientific decision picture that ties claims, follow-up posture, linked observed results, current support quality, governed support posture, current support coherence, governed support reuse, broader governed reuse, broader continuity, continuity-cluster posture, promotion-candidate posture, promotion-gate status, future reuse candidacy, trust tier, provenance confidence, governed review posture, session-family carryover review, manual-vs-derived governance origin, reviewer workflow attention, carryover guardrails, and a bounded predictive-path readiness summary together without implying autonomous planning. Those distinctions remain bounded support language rather than proof, planning, or autonomous scientific judgment.

What it is not yet is a true scientific discovery engine or a mature evidence-learning system. The main scientific weakness is that the core still relies on bounded tree-model scoring, similarity-shaped novelty/applicability heuristics, and a remaining legacy baseline fallback path. The predictive path is now more explicit than before: the product can distinguish bounded prioritization task semantics, representation limits, model-signal summaries, heuristic shortlist policy, governance gates, current evaluation coverage, and inspectable predictive failure modes. The current scoring layer is also stronger than it was: candidate rows now carry explicit raw predictive signal, raw-vs-heuristic blend weights, blended priority, representation-support adjustment, support-density context, bounded final priority score, score-decomposition summaries, and compact scoring failure-mode labels instead of forcing the user to read one opaque ordering number. Candidate-level uncertainty is now also more structured: the system can surface bounded uncertainty, fragility, neighbor-gap closeness, strong-vs-fragile-vs-thin signal status, and caution summaries rather than relying only on one generic uncertainty scalar. Representation support is also materially less thin than before because the live feature contract now uses a broader RDKit descriptor set plus similarity-density context, and model-family selection now compares bounded random-forest versus extra-trees candidates before saving the chosen session model. Offline ranking evaluation now also records candidate-separation, shortlist-stability, closeness-band, heuristic-influence, representation-sensitivity, calibration-awareness, model-family comparison, comparison-cohort, session-variation, and representation-support diagnostics so version-to-version scoring changes can be judged more honestly. But that path is still bridge-state. The representation layer is still descriptor-and-similarity shaped rather than mechanism-aware, the heuristic policy layer still does real shortlist work, and the offline evaluation harness is still bounded diagnostic scaffolding rather than a mature benchmark system. The main architectural weakness is improved but not solved: the system now defines explicit evidence-use boundaries, persists a selective evidence activation policy, records future-learning eligibility plus controlled activation boundaries, activates a very small amount of recommendation/ranking-context reuse, persists predictive-task and representation/evaluation contracts, and now persists first-class Claim objects, but older sessions and some read paths still depend on compatibility reconstruction from database metadata plus filesystem artifacts.

What must not be overstated: recommendations are not experimental truth, workspace memory is not yet the same as model learning, and broader product language should not imply the scientific core is already fully target-neutral or discovery-complete. Silent generic biodegradability defaults have been reduced, but backward-compatibility adapters and the legacy baseline bundle still remain.

The next strategic direction is no longer to widen the governance shell. It is now to continue the predictive-core / representation / evaluation rigor phase with stronger signal trustworthiness and better evaluation discipline. The product surface now separates evidence, model output, policy output, bridge-state fallback, and measurement-first ranking semantics more clearly; canonical scientific session truth now drives more comparison behavior; evidence records now carry explicit activation boundaries; the system can distinguish future-learning-eligible evidence from memory-only or permanently non-active evidence; a narrow, surfaced form of recommendation/ranking-context reuse is now live for continuity framing without changing model outputs; and the first Phase 3 scientific loop objects now exist as DB-backed bounded assertions, proposed next-experiment records, observed outcomes, support-change records, a target-scoped belief-state summary with explicit support-change governance, and a lightweight governed-review ledger that now preserves trust-tier, provenance-confidence, source-class, review-posture, promotion-gate, reversal history, manual reviewer attribution, concise reviewer-note context, reopen/revise traces, and supersession traces for claims, target-scoped belief-state posture, continuity-cluster posture, and session-family carryover posture. Numeric outcomes can now be surfaced more explicitly as decision-useful, limited, context-limited, or unresolved under the current basis; current support can now be surfaced as coherent, contested, degraded, or historically stronger than the present; broader continuity can now be surfaced as coherent, selective, contested, historical-heavy, or absent; broader reuse can now be discounted when the active record set is mixed, degraded, contradiction-limited, or mainly historical; continuity clusters can now be surfaced as local-only, context-only, selective, contradiction-limited, historical-heavy, or promotion-candidate quality; promotion boundaries can now be surfaced with explicit stability, gate, denial, downgrade, and quarantine semantics; local-only-by-default trust boundaries now explicitly resist volume-driven influence; the reviewer workflow now adds a bounded governance inbox, item prioritization, richer reviewer attribution context, concise rationale visibility, and cross-session carryover inspection without hiding the derived posture; and the predictive path now persists explicit bounded task contracts, richer representation summaries, stronger raw-vs-heuristic score semantics, bounded uncertainty semantics, fragility diagnostics, calibration-aware caution, model-family comparison traces, offline separation/stability/closeness diagnostics, and session-variation comparison summaries so the project can see which parts are raw predictive signal, which are heuristic shortlist policy, which are representation-support adjustments, where rankings are too close to separate honestly, and where governance still changes final usefulness. Those labels still remain bounded-support language rather than calibration, live learning, deep contradiction ontology, or scientific adjudication. The next step is now to strengthen deeper representation quality and more principled evaluation sets on top of this somewhat stronger engine rather than to keep widening shell semantics.
