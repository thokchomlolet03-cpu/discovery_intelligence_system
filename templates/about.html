Method and limitations Interpretation guide

What the system means, and what it still does not mean.

Discovery Intelligence is a session-backed decision-support system for helping researchers prioritize molecules for testing, review, and learning. It does not replace experiments, domain expertise, or scientific accountability.

Best use
  • Compare candidate sets before limited wet-lab capacity is spent.
  • Inspect novelty, uncertainty, and confidence together instead of separately.
  • Use the workbench to plan what to test next, not to claim experimental truth.

Current Pipeline

The system ingests molecular data, builds RDKit-derived features, trains or reuses a calibrated classifier, scores confidence and uncertainty, estimates novelty relative to known chemistry, and ranks molecules by experiment value and review priority.

Confidence

Confidence is the model’s estimated strength for a predicted label. Higher confidence can support screening, but it still does not guarantee real-world experimental behavior.

Uncertainty

Uncertainty indicates where the model is less sure. High-uncertainty molecules are often more useful for learning and review than for immediate, high-trust claims.

Novelty

Novelty reflects how dissimilar a molecule is from chemistry the system already knows. High novelty can be valuable for exploration, but it also means the model may be operating farther from familiar data.

Experiment Value

Experiment value is a prioritization signal that combines uncertainty, novelty, and confidence-related terms to suggest which molecules may be most useful to test next.

Appropriate Use

Use the system to prioritize review and testing queues, compare candidate lists, and identify where the model sees high-value follow-up work. It should support planning, not replace lab validation.

Session model

The web product now works through durable sessions.

Upload, Discovery, and Dashboard are designed to reopen saved artifacts, scientific summaries, and review state so a run can be returned to rather than disappearing after one page load.

Measurement-aware evidence

Quantitative scientific values matter.

When measurements such as pIC50 or other assay values are present, the system treats them as evidence about the dataset structure and can use them to support ranking diagnostics, label derivation, and interpretation.

Limitations

Predictions are model-guided recommendations, not experimental truth. Results should be used to prioritize testing, not replace experiments. Performance is strongest for chemistry similar to the system’s training data, and low-quality uploads or weak label coverage can reduce reliability.
