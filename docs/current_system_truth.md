# Current System Truth

Discovery Intelligence today is a session-aware molecular decision-support workbench. It lets users upload data, run prioritization workflows, review candidates, reopen sessions, compare runs, and carry forward workspace feedback. It is a real product surface with operational scaffolding, not just a modeling script.

What it is good at today is workflow continuity, reviewability, session persistence, decision traceability, and increasingly honest session rendering. The strongest current scientific truth is that it can produce recommendation lists from chemistry-based representations with explicit session context, target-aware caution signals, and clearer regression-vs-classification semantics. It now also creates first-class proposed claims, claim-linked proposed experiment requests, human-entered observed experiment-result records, bounded belief-update records, and a target-scoped BeliefState summary that tracks the current support picture without implying final truth or automatic learning. The strongest current product truth is that Upload, Discovery, Dashboard, and Sessions now behave like one connected workbench.

What it is not yet is a true scientific discovery engine or a mature evidence-learning system. The main scientific weakness is that the core still relies heavily on random-forest scoring, similarity-based novelty/applicability heuristics, and a remaining legacy baseline fallback path. The main architectural weakness is improved but not solved: the system now defines explicit evidence-use boundaries, persists a selective evidence activation policy, records future-learning eligibility plus controlled activation boundaries, activates a very small amount of recommendation/ranking-context reuse, and now persists first-class Claim objects, but older sessions and some read paths still depend on compatibility reconstruction from database metadata plus filesystem artifacts.

What must not be overstated: recommendations are not experimental truth, workspace memory is not yet the same as model learning, and broader product language should not imply the scientific core is already fully target-neutral or discovery-complete. Silent generic biodegradability defaults have been reduced, but backward-compatibility adapters and the legacy baseline bundle still remain.

The next strategic direction is to keep making the bridge-state scientific core more operationally explicit without breaking the workbench. The product surface now separates evidence, model output, policy output, bridge-state fallback, and measurement-first ranking semantics more clearly; canonical scientific session truth now drives more comparison behavior; evidence records now carry explicit activation boundaries; the system can distinguish future-learning-eligible evidence from memory-only or permanently non-active evidence; a narrow, surfaced form of recommendation/ranking-context reuse is now live for continuity framing without changing model outputs; and the first three new scientific loop objects, Claim, ExperimentRequest, and ExperimentResult, now exist as DB-backed bounded assertions, proposed next-experiment records, and observed outcome records. The next step is to add belief-update and belief-state tracking without letting those objects overstate certainty.
